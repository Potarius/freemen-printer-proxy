# Device Package Format

Documentation for the device configuration package generated by Freemen Provisioner.

## Overview

The device package is a self-contained folder with all configuration files needed to deploy Freemen Printer Proxy on a target device (Raspberry Pi or Linux server).

## Package Structure

```
output/<device-id>/
├── device.json                    # Device configuration (JSON)
├── device.env                     # Environment variables
├── docker-compose.yml             # Docker Compose configuration
├── docker-compose.override.yml    # Local overrides (optional)
├── setup.sh                       # Platform-specific setup script
├── cloudflared.service            # Systemd service (optional)
└── README.md                      # Setup instructions
```

## Files Description

### device.json

Main device configuration file containing all metadata.

```json
{
  "$schema": "https://freemen.io/schemas/device-v1.json",
  "version": "1.0.0",
  "generatedAt": "2024-01-15T10:30:00.000Z",
  "generatedBy": "Freemen Provisioner",
  
  "device": {
    "id": "fpp-abc123xyz",
    "name": "Office Printer Proxy",
    "platform": "raspberry-pi"
  },
  
  "cloudflare": {
    "accountId": "abc123...",
    "zoneId": "xyz789...",
    "zoneName": "example.com",
    "tunnel": {
      "id": "tunnel-uuid",
      "name": "freemen-printer-proxy"
    },
    "hostname": "printer.example.com",
    "publicUrl": "https://printer.example.com"
  },
  
  "service": {
    "port": 6500,
    "localUrl": "http://localhost:6500"
  },
  
  "printer": {
    "ip": "192.168.1.100",
    "port": 9100,
    "protocol": "jetdirect"
  }
}
```

### device.env

Environment variables for runtime configuration.

```env
# Device Identity
DEVICE_ID=fpp-abc123xyz
DEVICE_NAME=Office Printer Proxy

# Service Configuration
PORT=6500
API_KEY=${API_KEY:-change-me-in-production}

# Cloudflare Tunnel
TUNNEL_TOKEN=eyJhIjo...
TUNNEL_ID=tunnel-uuid
TUNNEL_NAME=freemen-printer-proxy
PUBLIC_HOSTNAME=printer.example.com
PUBLIC_URL=https://printer.example.com

# Printer Configuration
PRINTER_IP=192.168.1.100
PRINTER_PORT=9100
PRINTER_PROTOCOL=jetdirect

# Logging
LOG_LEVEL=info
```

### docker-compose.yml

Docker Compose configuration for running the printer proxy and cloudflared.

```yaml
version: '3.8'

services:
  freemen-printer-proxy:
    image: ghcr.io/potarius/freemen-printer-proxy:latest
    container_name: freemen-printer-proxy
    restart: unless-stopped
    env_file:
      - device.env
    ports:
      - "6500:6500"
    networks:
      - freemen-network

  cloudflared:
    image: cloudflare/cloudflared:latest
    container_name: freemen-cloudflared
    restart: unless-stopped
    command: tunnel --no-autoupdate run --token ${TUNNEL_TOKEN}
    env_file:
      - device.env
    networks:
      - freemen-network
    depends_on:
      - freemen-printer-proxy

networks:
  freemen-network:
    name: freemen-network
    driver: bridge
```

### setup.sh

Platform-specific setup script that:
1. Checks for root privileges
2. Detects architecture (amd64/arm64)
3. Installs Docker if not present
4. Installs Docker Compose if not present
5. Creates installation directory
6. Starts services

## Platform Support

### Raspberry Pi

- Architecture: `aarch64` (arm64) or `armv7l`
- Default user: `pi`
- Setup script: `setup.sh` (Pi-optimized)

### Linux (Ubuntu/Debian)

- Architecture: `x86_64` (amd64) or `aarch64` (arm64)
- Package managers: apt, dnf, yum
- Setup script: `setup.sh` (generic Linux)

## Deployment Options

### Option 1: Automated Setup (Recommended)

```bash
# Copy package to device
scp -r output/fpp-abc123xyz pi@raspberrypi.local:~/

# SSH to device
ssh pi@raspberrypi.local

# Run setup
cd fpp-abc123xyz
sudo chmod +x setup.sh
sudo ./setup.sh
```

### Option 2: Manual Docker Setup

```bash
# Navigate to package
cd fpp-abc123xyz

# Start services
docker compose up -d

# View logs
docker compose logs -f
```

### Option 3: Systemd Service (No Docker)

```bash
# Install cloudflared manually first
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64 -o /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared

# Install service
sudo cp cloudflared.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable cloudflared
sudo systemctl start cloudflared
```

## Security Notes

⚠️ **Important:**

1. **Never commit to version control** - The package contains sensitive tunnel tokens
2. **Treat tunnel token like a password** - It grants access to your Cloudflare tunnel
3. **Change default API key** - Update `API_KEY` in `device.env` before production
4. **Set proper permissions** - `device.env` should be `chmod 600`

## Integration with Runtime

The generated package is designed to work with the existing Freemen Printer Proxy runtime:

1. **Environment variables** are read from `device.env`
2. **Tunnel token** is passed to cloudflared via `TUNNEL_TOKEN`
3. **Port** is configurable via `PORT` (default: 6500)
4. **Printer settings** are read from `PRINTER_*` variables

## Package Generation

Packages are generated by the `DevicePackageGenerator` service:

```typescript
import { DevicePackageGenerator } from './services/device-package-generator';

const generator = new DevicePackageGenerator('./output');
const pkg = generator.generatePackage({
  deviceId: 'fpp-abc123xyz',
  deviceName: 'Office Printer Proxy',
  platform: 'raspberry-pi',
  hostname: 'printer.example.com',
  tunnelName: 'freemen-printer-proxy',
  tunnelToken: 'eyJhIjo...',
  tunnelId: 'tunnel-uuid',
  accountId: 'abc123...',
  zoneId: 'xyz789...',
  zoneName: 'example.com',
  servicePort: 6500,
  printerIp: '192.168.1.100',
  printerPort: 9100,
});
```

## Output Location

Default output directory:
- **Tauri app**: `Documents/Freemen Provisioner/output/<device-id>/`
- **Browser mode**: Downloaded as self-extracting shell script
